# Exec Plans Index

- [active/](active/)

- [execution-index.md](execution-index.md)

## 规则

- 正在进行的复杂任务放在 `active/`
- 完成的计划移到 `completed/`
- `spec-team` 会自动创建 active 执行计划
- `spec-finish` 会自动归档对应计划
- `execution-index.*` 记录关键执行事件，便于长期检索
- `execution-by-status.md` 按状态聚合最近执行
- `execution-by-slug.md` 按需求 slug 聚合最近执行
