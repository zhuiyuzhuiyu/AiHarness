# DESIGN

## 目标

AiHarness 的目标是把需求接入、规格生成、执行、评审、验证和交付收尾组织成一套 agent-first 的工程流水线。

## 设计原则

- 人类负责意图、边界和验收标准。
- agent 负责执行、回归、review 和文档落盘。
- 文档必须对 agent 可读，而不只是对人类可读。
- 每个复杂任务都应该有可追踪的 spec 和执行计划。

## 当前架构层

- `skills/`: 任务能力与规则
- `commands/`: 显式工作流入口
- `hooks/`: 自动门禁
- `scripts/`: 共享编排逻辑
- `specs/`: 需求与交付工件
- `docs/`: 长期知识与记录系统
