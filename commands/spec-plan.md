# /spec-plan

## Purpose

Generate executable implementation slices in `tasks.md`.

## Inputs

- `requirements.md`
- `design.md`

## Outputs

- `specs/<initiative-slug>/<YYYY-MM-DD>-v1/tasks.md`

## Rules

- Tasks must be independently reviewable
- Tasks must have a validation command
- Each task should identify the best agent or model role
- Blockers and risks must be explicit
