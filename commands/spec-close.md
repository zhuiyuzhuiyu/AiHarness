# /spec-close

## Purpose

Finalize a completed change, update docs, and produce `handoff.md`.

## Inputs

- Completed implementation
- `review.md`
- `test-report.md`

## Outputs

- Updated project docs
- `specs/<initiative-slug>/<YYYY-MM-DD>-v1/handoff.md`

## Rules

- Update only docs that truly drifted
- Summarize delivered behavior and residual risk
- Flag deferred work explicitly
